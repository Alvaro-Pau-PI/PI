# Iteración 1: Entorno, escaparate y contacto (18 horas)

Esta primera iteración se ha centrado en establecer las bases esenciales para el desarrollo del proyecto de E-commerce, con los siguientes objetivos y actividades realizadas:

## Objetivos principales

- Configurar los entornos de desarrollo y servidor, tanto local como remoto, para garantizar que el trabajo sea reproducible y estable.
- Crear y gestionar el tablero de proyectos con metodología ágil (GitHub Projects / Trello) para organizar, planificar y hacer seguimiento de las tareas.
- Elaborar un cronograma inicial con GanttProject para establecer la planificación temporal y dependencias entre tareas.
- Asignar roles y tareas dentro del equipo para distribuir la responsabilidad de manera clara y eficiente.
- Identificar y documentar los riesgos laborales y las medidas de prevención para garantizar la seguridad y el bienestar durante el trabajo.
- Desarrollar la página inicial con un formulario de contacto funcional y validado, asegurando que la interacción básica con el usuario sea correcta.

## Actividades realizadas

### Configuración de entorno

- Instalación y configuración de herramientas necesarias (IDE, Docker, Git).
- Creación y documentación del entorno local y remoto para todo el equipo.
- Definición de la estructura básica del proyecto con carpetas `frontend/`, `backend/`, `docs/`, etc.

### Gestión ágil del proyecto

- Creación del tablero Kanban en GitHub Projects o Trello.
- Definición de columnas básicas: To Do, In Progress, Done.
- Creación y asignación de tarjetas/tareas con responsables y etiquetas.

### Planificación temporal

- Creación del cronograma visual con GanttProject.
- Definición de tareas, duración estimada y dependencias.
- Exportación del cronograma y más adelante su incorporación a la documentación.

### Asignación de roles

- Definición y documentación de los roles Scrum Master, Developer y Documentador.
- Reparto de las tareas concretas y reevaluación periódica para asegurar el equilibrio.

### Prevención de riesgos laborales

- Identificación de los riesgos físicos, ambientales y psicosociales.
- Redacción del documento de riesgo laboral y medida preventiva.
- Compromiso del equipo en mantener un entorno de trabajo seguro y saludable.

### Desarrollo del formulario inicial

- Creación del formulario con los campos: nombre, email, ciclo formativo, teléfono y consentimiento de datos.
- Desarrollo e implementación de las validaciones básicas en el cliente (próximamente JS) y servidor con PHP.
- Integración del formulario en la página web del proyecto.

## Resultados obtenidos

- Un entorno de trabajo completamente configurado y documentado.
- Un tablero Kanban operativo que permite el seguimiento de todas las tareas e incidencias.
- Un cronograma visual que guía la planificación futura del proyecto.
- Roles y responsabilidades claramente definidos y registrados.
- Un plan de riesgos laborales consensuado, implementado y documentado.
- Una página inicial con el formulario funcional y validaciones básicas realizadas en el backend.

---

Esta iteración establece las bases para las siguientes fases del proyecto con la metodología ágil como eje principal y prepara el terreno para el desarrollo y despliegue efectivos.
